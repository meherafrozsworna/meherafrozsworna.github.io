Research and Recent Works

Efficient Approaches to Speed Up Quantifying Transcript Abundance from RNA-seq Data

Co-Authors: Sadia Tasnim, Dr. Atif Hasan Rahman

Kallisto is an RNA sequencing quantification program that uses pseudoalignment for compatibility test between reads and reference transcriptome without complete alignment. For this, it is faster than other existing tools as well as accurate. After pseudoalignment, Expectation-Maximization algorithm is run to get the abundance estimate by adjusting latent parameters. We have done some modifications in the quantification process so that the EM algorithm would take fewer iterations to converge. So, the whole abundance estimation will cost less time and Kallisto will perform even faster. We have run our modified Kallisto on real datasets as well as idealized datasets.

Our approach to initialize the parameters to improve the EM Algorithm, that ensures less iterations in convergence.

Off-targets minimization in CRISPR-cas9 genome editing using Graph Transformer

Supervised By: Dr. A. B. M. Alim Al Islam

CRISPR (clustered regularly interspaced short palindromic repeats) is a family of DNA sequences found in the genomes of prokaryotic organisms such as bacteria and archaea and Cas9 is an enzyme that uses CRISPR sequences as a guide to recognize and cleave specific strands of DNA that are complementary to the CRISPR sequence. Off-target genome editing refers to nonspecific and unintended genetic modifications that can arise through the use of engineered nuclease technologies, that can produce potentially misleading and non-reproducible results. For off-targets prediction, Graph Transformer- a transformer neural network that can operate on arbitrary graphs, is used here.

Decentralized Federated Continual Learning

Supervised By: Dr. Muhammad Abdullah Adnan

In this project, standard continual learning learns on a sequence of tasks in each client machine. This continual learning is extended into a multiple client decentralized federated learning setting where a central global server won’t be required to accumulate the local gradient updates for the global model. Thus, the chance of single point failure is reduced and the computational power is given to clients.
